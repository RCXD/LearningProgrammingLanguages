큐
	FIFO(First In First Out) - 선입선출
	삽입과 삭제가 서로 다른 입구에서 일어난다
	한쪽에서는 삽입, 반대쪽에서는 삭제

	장점 : 시간 순서대로 처리해야할시 편리
	단점 : 삽입과 삭제하는 방향이 정해져있다

	front : 삭제
	rear : 삽입

선형큐
	[] 이용, 고정크기
원형큐
	[] 이용, 고정크기, 삭제시 다시 빈공간 활용 가능
연결큐
	rear 변수에 추가할때마다 newNode로 바꾸어주기
	첫추가가 아닌경우에는 현재 rear의 next에 newNode를
	연결해 주는것이 중요하다
	삭제시에는 front 위치의 데이터를 가져오고, front를
	다음 노드로 이동
